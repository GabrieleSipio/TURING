FAT (File Allocation Table):
il file system FAT adotta come tecnica di allocazione dei file la lista linkata.  Questo perché è semplice, facile da implementare e ancora ampiamente utilizzata; questa lista linkata, però, a differenza di alcuni file system ancora più primitivi, è memorizzata all’interno di una tabella che si chiama FAT. FAT è sia il nome dell’intero file system, sia il nome della struttura dati che contiene le liste linkate dei blocchi di tutti i file presenti nel disco. Viene fatta questa distinzione perché quando si parla di file system non basta parlare della tabella che gestisce i blocchi allocati ai file, bisogna anche parlare di come sono organizzate le directory, di come è organizzata la lista dei blocchi liberi, di come viene fatta l’allocazione e via discorrendo. La FAT è un array di puntatori, tanti quanti sono i blocchi del disco. Il file system FAT, quando gli viene detto di allocare un file, va a scorrere la lista dei blocchi liberi, il primo blocco libero che trova lo utilizza senza preoccuparsi di nessun aspetto di località ed è questo il motivo per il quale i file nel file system FAT possono essere distribuiti in maniera così arbitraria ed è per questo motivo che le prestazioni possono degradare selvaggiamente ed è per questo che serve la deframmentazione, per ricompattare i blocchi dei file. In generale, l’elemento all’inizio della lista linkata, contiene il puntatore a quello successivo e così via per ogni elemento successivo, fino a quando non si arriva all’interno dell’ultimo elemento dove ci sarà scritto -1 (oppure 0, insomma un valore non utilizzabile) in maniera tale che chi scorre la lista linkata sabbia che quello è l’ultimo blocco del file. I vantaggi sono che p facile trovare un blocco libero; in realtà i blocchi liberi sono memorizzati in questa tabella stessa marcati come liberi, quindi il valore contenuto per i blocchi liberi sarà un valore specifico che indica che quel blocco è libero. Se devo allocare un nuovo file e mi servono 4 blocchi liberi, scandisco la FAT dall’inizio, trovo che i primi 4 posti liberi sono utilizzabili per quel file e quelli li alloco al file e di conseguenza li leggo in una lista linkata. In questo modo è facile trovare un blocco libero, è facile aggiungere informazioni ad un file, se io voglio far cresce un file, trovo degli altri blocchi liberi, li collego alla lista linkata e in questo modo allungo il file liberamente ed è anche facile cancellare un file, basta marcare  come liberi tutti quanti i suoi blocchi nella lista linkata. I punti di debolezza sono il fatto che la FAT debba stare obbligatoriamente nel disco, in particolare, nel disco, i primi blocchi sono riservati per scopi particolari, mentre i blocchi successivi contengono la FAT e devono stare lì perché all’interno sono contenute informazioni critiche: se si perde la FAT, si perde il contenuto del disco, le informazioni ci sono ancora ma  non si ha più un ordine. Altresì non è un modo efficiente di usare il disco, per questo motivo in realtà la FAT, quando voi inserite la pendrive usb, come prima cosa viene caricata in memoria principale, facendo questo, tutte le operazioni sulla FAT si fanno in memoria quindi sono velocissime e a questo punto leggere il file mi comporta soltanto la lettura dei blocchi effettivi del file, niente di più, niente di meno. Il problema è che, se la FAT normalmente sta sul disco ma quando il computer è acceso deve stare in memoria principale, quanto è grande? La dimensione della FAT è un problema in quanto la FAT in memoria me la devo tenere tutta, indipendentemente dai file che voglio effettivamente leggere o scrivere, perché non so dove sono collocati a priori, quini comunque me la dovrò caricare tutta. Poi altri problemi piuttosto gravi sono il fatto che nel file system FAT non ho metadati, ho poche informazioni sui file ed in particolare non ho informazioni di protezione , questo p un limite molto forte, d’altra parte la FAT era pensata per un personal computer ad uso personale, non si pensava che un utente dovesse proteggersi da se stesso, nei sistemi attuali, invece, si, gli utenti vanno anche protetti da se stessi, questo è un limite grosso della FAT e l’altro limite grosso è legato alla frammentazione, non fa niente per cercare di mantenere i file compatti, di tenere i blocchi allocati ai file compatti, e questo vuol dire che allocando a caso i blocchi le prestazioni possono peggiorare in maniera drammatica.
